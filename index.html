프로젝트 소개

사용자 식별 정보를 AI에 전달하지 않고, 대화 종료 시 매핑 데이터를 즉시 휘발시키는 '비존재(비식별화)' 미들웨어 기반 프라이버시 보호 AI 1:1 채팅 앱 MVP를 Flutter(Android)·FastAPI·Redis로 60일 내 구축하고 구글 플레이스토어 등록까지 완료합니다.

프로젝트 개요

핵심 요구사항 분석

  • 비존재(비식별화) 미들웨어 — 실제 User ID 대신 일회성 난수 Session ID로 OpenAI API 통신, 세션 종료 시 'User ID ↔ Session ID' 매핑 즉시 파기
  • 데이터 휘발(TTL) — Redis 기반 임시 매핑 저장 및 TTL 만료/명시적 삭제로 매핑 데이터가 서버에 절대 잔존하지 않도록 보장
  • 회원 및 인증 — 이메일 기반 간편 회원가입·로그인, 계정 탈퇴 시 DB 내 사용자 데이터 완전 파기
  • AI 1:1 채팅 — OpenAI API 연동 채팅방 생성, 텍스트 기반 실시간 채팅 UI, 비동기 처리로 응답 대기 최소화
  • 1차 개인정보 마스킹 — 클라이언트 단 정규식(Regex) 기반 전화번호 등 개인정보 1차 마스킹
  • 관리자 백오피스 — 전체 가입자 수 조회, OpenAI API 토큰 사용량 모니터링
  • 스토어 등록 — 구글 플레이스토어 심사 및 등록 지원 (iOS 제외)

제안 포인트

1. '비존재' 미들웨어를 독립 계층으로 설계합니다. 인증·채팅 로직과 분리된 별도 미들웨어 계층에서 세션 난수화(Session ID 발급), 매핑 저장(Redis), TTL 휘발을 일관되게 처리합니다. 클라이언트→백엔드→OpenAI 전 구간에서 실제 User ID가 AI 측에 전달되지 않도록 토큰 경계를 명확히 분리합니다.

2. 데이터 휘발을 다중 안전장치로 보장합니다. Redis TTL(시간 만료)에만 의존하지 않고, ① 대화방 이탈·세션 종료 이벤트 시 즉시 명시적 삭제, ② TTL 백업 만료, ③ 계정 탈퇴 시 DB·캐시 동시 파기의 3중 구조로 '매핑 데이터 잔존 0'을 설계합니다.

3. AI 응답 대기 시간을 비동기로 최소화합니다. FastAPI의 async 구조와 OpenAI 스트리밍 응답을 활용해 첫 토큰 도달 시간을 단축하고, 채팅 UI는 점진적 렌더링으로 체감 속도를 높입니다.

4. 유사 프로젝트 수행 경험으로 일정을 안정적으로 준수합니다. Flutter 멀티플랫폼 앱 출시, OpenAI 연동 AI 분석, E2E 암호화 기반 프라이버시 설계를 직접 수행한 경험을 보유하여, 착수 2주 내 화면 설계·아키텍처 확정, 6주 내 개발 완료라는 마일스톤을 현실적으로 달성합니다.

PRD (Product Requirements Document)

프로젝트 요구사항을 정의하고 기능 범위를 명확히 합니다.

1. 프로젝트 배경 및 목적

생성형 AI 사용 시 발생할 수 있는 개인정보 유출 및 데이터 학습 우려를 원천 차단하는 것이 목표입니다. 사용자의 식별 정보를 AI에게 넘기지 않고, 대화 종료 시 관련 데이터를 즉각 휘발시키는 '비존재(비식별화)' 미들웨어를 구축하여, AI가 사용자를 전혀 특정할 수 없도록 연결 고리를 차단한 AI 1:1 채팅 앱의 MVP 버전을 개발합니다.

2. 사용자 정의

사용자 유형설명주요 기능
일반 사용자프라이버시 보호 환경에서 AI와 대화하는 이용자회원가입/로그인, AI 1:1 채팅, 계정 탈퇴
관리자서비스 운영 현황을 모니터링하는 운영자가입자 수 조회, OpenAI 토큰 사용량 모니터링

3. 기능 요구사항

3.1 회원 및 인증

  • [필수] 이메일 기반 간편 회원가입 및 로그인
  • [필수] 계정 탈퇴 시 DB 내 사용자 데이터 완전 파기

3.2 AI 1:1 채팅

  • [필수] OpenAI API 연동 AI 1:1 채팅방 생성 및 대화
  • [필수] 텍스트 기반 실시간 채팅 UI
  • [필수] 비동기 처리로 AI 응답 대기 시간 최소화

3.3 비존재(비식별화) 미들웨어

  • [필수] 세션 난수화 — 실제 User ID가 아닌 일회성 난수 Session ID로 AI API 통신
  • [필수] 데이터 휘발 — 대화방 이탈/세션 종료 시 Redis 임시 매핑('User ID ↔ Session ID') 즉시 파기
  • [필수] 클라이언트 단 정규식(Regex) 기반 1차 개인정보(전화번호 등) 마스킹

3.4 관리자 백오피스

  • [필수] 전체 가입자 수 단순 조회
  • [필수] OpenAI API 토큰 사용량 모니터링
